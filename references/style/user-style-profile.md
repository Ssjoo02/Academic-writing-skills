# User Style Profile

Use this when the user provides previous papers they wrote.

## Learn

- Section organization: observe the order of motivation, related work, method, evidence, analysis, and limitations; note whether the user prefers front-loaded contributions, gradual setup, or early technical framing.
- Paragraph density: observe typical paragraph length, number of ideas per paragraph, transition frequency, and whether paragraphs emphasize exposition, comparison, or evidence.
- Technical detail level: observe how much implementation detail, formal notation, algorithmic description, setup detail, and error analysis the user usually includes before asking readers to accept a claim.
- Terminology habits: observe preferred names for methods, datasets, baselines, assumptions, and evaluation concepts; preserve consistent technical vocabulary while avoiding repeated distinctive phrasing.
- Claim caution level: observe whether the user states findings as definitive, bounded, comparative, preliminary, or hypothesis-generating; match the caution level to available evidence.
- Figure/table rhythm: observe where figures and tables usually appear, whether they introduce concepts or support claims, and how often quantitative evidence interrupts narrative text.
- How limitations are framed: observe whether limitations are presented as scope boundaries, future work, threats to validity, failure modes, or reviewer-facing risk controls.

## Do Not Learn

- Distinctive phrases.
- Sentence-level wording.
- Paragraph wording.
- Idiosyncratic expressions that could imitate the author too closely.

## Output

Summarize user style as:

- structure preference,
- density preference,
- terminology rules,
- claim strength tendency,
- figure/table preference,
- forbidden imitation notes.
